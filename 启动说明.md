
## 启动步骤

### 1. 启动后端（zhice-server）

```bash
cd D:\科研\医保大模型\workspace_MIL\zhice-server
npm run dev
```

**预期输出：**
```
✅ 智策医保中间层运行在 http://localhost:3001
📌 默认管理员: 13800000000 / admin123
```

**如遇端口冲突：** 修改 `.env` 文件中的 `PORT=3001` 改为其他端口（如 `PORT=3002`），同时同步修改 `zhice-frontend/vite.config.ts` 中的 proxy target 端口。

**后端依赖环境变量（`.env` 文件）：**

| 变量 | 说明 | 示例 |
|------|------|------|
| `PORT` | 后端端口 | `3001` |
| `JWT_SECRET` | JWT 加密密钥 | `zhice-secret-key-2025` |
| `DIFY_API_BASE` | Dify API 地址 | `https://llmanswer.seu.edu.cn/v1` |
| `DIFY_API_KEY` | Dify API 密钥 | `app-xxxxxxxxxxxxxxxx` |
| `DIFY_USER_ID` | Dify 系统用户标识 | `zhice-user` |

> `.env` 文件已创建好，只需修改 `DIFY_API_KEY` 和 `DIFY_API_BASE` 为实际值。

### 登录
1. 打开 `http://localhost:5173`
2. 输入手机号 `13800000000`，密码 `admin123`
3. 点击登录，应跳转到对话页面

2. 等待流式回答返回（回答框随内容自动变大）

## 常见问题

**Q: 后端报 `EADDRINUSE` 端口被占用？**
A: 先查占用进程 `netstat -ano | findstr :3001`，然后 `taskkill //F //PID <进程号>`，再重新启动。

**Q: 对话没有响应？**
A: 检查 `.env` 中的 `DIFY_API_BASE` 和 `DIFY_API_KEY` 是否正确，以及 Dify 服务是否可达。

**Q: 验证码一直提示错误？**
A: 当前验证码为模拟模式，任何手机号统一填 `123456` 即可。如需对接真实短信，修改后端 `src/routes/auth.ts` 中的 `send-code` 接口。

**Q: 前端页面样式异常？**
A: 确保安装了依赖：`cd zhice-frontend && npm install`。

**Q: 如何更换聊天页Logo？**
A: 将 Logo 图片放到 `zhice-frontend/public/logo.png`（推荐 48×48 或 96×96 PNG，支持透明背景）。如果不上传，会显示默认的"智"字文字Logo。

**Q: 如何更换聊天页背景图？**
A: 将图片放到 `zhice-frontend/public/bg-chat.png`。

**Q: 模型回答显示Markdown原文而不是格式化内容？**
A: 确保安装了 react-markdown：`cd zhice-frontend && npm install`。前端已自动渲染 Markdown 为标题、表格、代码块等格式。

**Q: 注册时没有验证码输入框了？**
A: V2 版本已取消手机验证码步骤，改为设置密码。注册时填写密码和确认密码即可，手机验证步骤仅用于确认手机号。
